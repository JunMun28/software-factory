Executive thesisThe New SDLC With Vibe Coding

The center of gravity moved from syntax to intent

AI now writes large chunks of code. The scarce work is specifying direction, feeding context, verifying behavior, and deciding what is safe to ship.

What changes

  • Developers express goals, constraints, examples, and acceptance bars.
  • Agents translate that intent into code, tests, diffs, plans, and reviews.
  • Teams build reusable harnesses instead of treating every prompt as a one-off.

What does not disappear

  • Requirements and architecture still need human judgment.
  • Verification becomes more important, not less.
  • Engineering culture is amplified: strong teams scale, weak habits scale too.

Role shiftThe New SDLC With Vibe Coding

Developers move between conductor and orchestrator modes

The paper names two common postures: hands-on pairing in the IDE and async delegation to background agents.

Conductor

Works in real time with an IDE or terminal agent.

  • steers line by line
  • checks assumptions quickly
  • keeps local context tight

Orchestrator

Splits work across async agents and reviews outputs.

  • writes crisp specs
  • decomposes tasks
  • evaluates and merges safely
The last 20 percent is where integration, taste, and domain judgment show up.

Adoption playbookThe New SDLC With Vibe Coding

Start by making one workflow repeatable and measurable

The paper's advice is deliberately practical: build context as code, set an eval bar, and distinguish prototypes from production.

Individuals

Create AGENTS.md; write skills; pick one repetitive workflow; add tests before agent code; review every shipping line.

Leaders

Treat context as code; reward eval quality; reshape review; fund harness assets; separate demo speed from production readiness.

Organizations

Invest in substrate; adopt interop protocols; build hybrid teams; hire for judgment, decomposition, and evaluation.

The edit loop

SANDBOX · APP-PREVIEW DESIGNED · BUILDS NEXT
  1. aSeed. app-preview clones the request's repo at the exact previewed commit, and proves it green before you type a word.
  2. bTalk and point. Chat with the agent, or click any element and say what to change. The app hot-reloads while the agent works.
  3. cGreen turns become Versions. Every green turn is its own git commit, a checkpoint you can restore or revert, like v0. Red work is never kept.
  4. dSend back. Your Versions land on the request's branch as your own commits, and the spec records what changed: POST …/preview/import-edit
↑ Rejoins the line at 03 · The gate: gate and review re-run on your commits; the build stage is skipped, the code already exists. A red gate rejects the import and you fix it right here.

One road to production

Every path (feedback rounds, live sandbox edits) ends in the same merge and deploy. There is no side door.

Trust is not transitive

The sandbox gates its own turns, but the factory still re-runs its gates on anything that comes back.

Nothing ships without you

Accept is a human gate. The line holds at 06 until you decide.

Stations 01–08 run today (preview rounds behind FACTORY_PREVIEW). The sandbox edit loop is the ng-v0 bridge design (D3, docs/design/ng-v0-bridge.md): seed, export, and import-edit are the three pieces left to build.

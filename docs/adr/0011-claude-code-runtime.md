# Claude Code is the first real agent runtime; enforcement stays at the artifact layer

**Status:** accepted
**Amends:** ADR 0001 (Copilot-first) · ADR 0009 (prototype seams)

ADR 0001 chose GitHub Copilot CLI as the primary runtime — but this team has Claude
Code and not Copilot. Because ADR 0001 deliberately placed enforcement at the
**orchestration/git layer** ("runtime-agnostic and checked against the Artifact, not
the agent's good behavior"), swapping the runtime costs nothing in guarantees: the
gates don't care which agent did the work.

Two opt-in modes wire Claude Code in behind the existing seams (`claude_exec.py` is
the single CLI chokepoint):

- **`FACTORY_BRAIN=claude`** — the Stage 1 intake brain. The interview questions and
  the grounded draft spec are generated by `claude -p` (JSON output, one turn, read-only
  tools disallowed). Every call degrades to the ScriptedBrain on error/parse failure —
  the interview is enrichment, never a blocker.
- **`FACTORY_RUNNER=claude`** — Stages 2–5 run for real. On Spec approval each Request
  gets a git workspace copied from `sample/`; the stage agents run headless Claude Code
  inside it and **every gate is machine-checked on the artifact**:
  architecture → `PLAN.md` exists · RED → pytest fails with cleanly-collected assertion
  failures (rc 1, not 2) · GREEN → suite passes **and the `tests/` tree hash is
  byte-identical to the RED snapshot** (the test-isolation gate, enforced for real —
  a cheating implementer's test edits are reverted and the item escalates) ·
  review → `REVIEW.md` exists, then the **human merge gate** is raised. Approving the
  merge runs `git merge work/<ref> → main` in the workspace — the local stand-in for
  the protected-branch promotion of ADR 0005.

## Bounded autonomy & escalation (CONTEXT.md held)

Each stage runs with `--max-turns` and a wall-clock timeout; any gate failure or
timeout **escalates (needs_human) with the reason — no automatic retry**. The
Recovery actions (Retry, Cancel) operate on escalated items exactly as before.

## Consequences

- Copilot remains compatible by design — a future `copilot_runner.py` would implement
  the same Executor seam; nothing else changes. ADR 0001's enforcement-placement
  decision is what made this a swap instead of a rewrite.
- The simulator stays the default (`FACTORY_RUNNER=sim`) so `make verify` and CI remain
  deterministic and free. Runner tests inject a fake executor and assert the FACTORY's
  guarantees (gates, isolation, escalation), never the model's behavior.
- Claude calls cost real tokens/time; the model is pinned small by default
  (`FACTORY_CLAUDE_MODEL`, haiku) and stage timeouts via `FACTORY_STAGE_TIMEOUT`.
